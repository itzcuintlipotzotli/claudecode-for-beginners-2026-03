What is Claude Code?

Conversational AI assistant (claude.ai / API) CLI agent built on top of Claude
You describe problems in chat Claude reads, edits and runs code directly in your project

Configuration & Permissions

Claude Code is configured at two levels:

  • CLAUDE.md — project instructions Claude reads every session: conventions, build commands, architecture notes. Run /init to generate one.
  • .claude/settings.json — controls tool permissions and environment variables, per-project or user-wide.

Permissions use allow and deny lists to decide which tools Claude can invoke without asking first.

Why safe configuration matters

Overly permissive settings let Claude modify or delete files and push code without prompting. Start restrictive and widen only what you need.

Sandboxing

Applies to participants running Claude Code locally via the CLI.

Sandboxing restricts the commands Claude Code can run to a defined set of filesystem paths and network hosts, reducing the risk of unintended changes outside your working directory.

Run /sandbox inside Claude Code to enable it. On macOS it works out of the box; Linux and WSL2 require additional packages.

Why sandboxing matters

Without sandboxing, a misunderstood instruction could overwrite unrelated files or make unintended network calls. Treat it as a safety net, especially when working with unfamiliar codebases.

Extending Claude Code

Claude Code can be extended in two ways:

  • Skills — custom slash commands (/your-command) defined as prompt templates in .claude/skills/. Use them to encode repeatable workflows such as /commit, /review, or project-specific tasks.
  • MCP Plugins — the Model Context Protocol (MCP) lets Claude connect to external tools and data sources (databases, APIs, services) as if they were native tools. Configure servers under the mcpServers key in settings.json.
